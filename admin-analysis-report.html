! Kritik Sorunlar

1 $plan->price / $plan->currency — Kaldırılmış kolon erişimi Kritik
Dosyalar: admin/users/create.blade.php ve admin/users/edit.blade.php

Çoklu para birimi güncellemesiyle price, price_yearly ve currency kolonları kaldırılıp prices JSON kolonu ile değiştirildi. Ancak kullanıcı oluşturma/düzenleme formlarında hala eski kolonlara erişiliyor.

Sonuç: Runtime hatası verir veya null döner. $plan->getDisplayPrice() veya benzeri metot kullanılmalı.
2 StoreUserRequest — super_admin rolü atanabilir Kritik
View'da super_admin filtreleniyor ama validation'da engellenmiyor. Crafted POST isteği ile super_admin rolü atanabilir.
// Mevcut (güvenlik açığı): 'role' => ['required', 'string', 'exists:roles,name'], // Olması gereken: 'role' => ['required', 'string', 'exists:roles,name', Rule::notIn(['super_admin'])],
3 UserService::createUser — Plaintext şifre loglanıyor Kritik
Exception durumunda Log::error('...', ['data' => $data]) ile ham şifre log dosyasına yazılıyor. $data array'i hash'lenmemiş parolayı içerebilir.
// Mevcut: Log::error('User creation failed', ['error' => $e->getMessage(), 'data' => $data]); // Olması gereken: Log::error('User creation failed', [ 'error' => $e->getMessage(), 'data' => Arr::except($data, ['password', 'password_confirmation']) ]);

  • admin/languages/ view dosyaları (yok)

~ Orta Öncelikli Sorunlar

7 RestaurantPolicy — create() metodu eksik Orta
RestaurantPolicy'de viewAny, view, update, delete var ama create() yok. StoreRestaurantRequest::authorize() → return true.
8 RestaurantController::destroy — authorize() çağrısı yok Orta
RestaurantPolicy::delete() mevcut ama controller'da hiç çağrılmıyor. Policy bypass ediliyor. Route middleware tek koruma katmanı.
9 Dashboard — 28 hardcoded çeviri Orta
dashboard.blade.php dosyasında 28 yerde app()->getLocale() === 'tr' ? '...' : '...' ternary kullanılıyor. __() ile lang dosyasından çekilmeli. 3. dil eklendiğinde kırılır.
10 Sidebar section label'ları hardcoded Orta
"İçerik/Content" ve "Sistem/System" etiketleri __() yerine inline ternary ile yazılmış. 3. dil eklenirse sadece Türkçe/İngilizce gösterir.
11 Header "Super Admin" etiketi hardcoded Orta
admin-header.blade.php satır 51'de "Super Admin" sabit İngilizce metin. __('admin.users.roles.super_admin') kullanılmalı.

↓ Düşük Öncelikli Sorunlar

12 DashboardController — Controller extend ediyor Düşük
Diğer tüm admin controller'lar BaseController extend ederken DashboardController plain Controller extend ediyor. Tutarsızlık.
13 DashboardController — try-catch yok Düşük
DB sorguları exception handling olmadan çalışıyor. Database hatası durumunda unhandled 500 hatası verir.
14 ImpersonateController::start — Dead code Düşük
Satır 41-43'te $user->hasRole('super_admin') kontrolü var ama aynı kontrol satır 24-26'da zaten return ediyor. Satır 41-43 ulaşılamaz (dead code).
15 togglePublish / toggleActive — try-catch yok Düşük
PostService::togglePublish() ve BlogCategoryService::toggleActive() doğrudan save() yapıyor. Hata durumunda exception loglanmıyor.
16 BlogCategoryController::reorder — Inline validation Düşük
FormRequest yerine controller'da $request->validate() kullanılıyor. CLAUDE.md'deki "FormRequest zorunlu" kuralına aykırı.
17 Orphaned dosya — posts/partials/form.blade.php Düşük
posts/partials/form.blade.php hiçbir view'dan include edilmiyor. Eski flat $post->slug yapısını referans ediyor (artık JSON). Silinmeli.
