Introduction

Public menu API for QR Menu SaaS. Fetch restaurant menus, categories and items to build your own integrations.

Base URL: http://qrmenu.test

Welcome to the QR Menu API. This API lets you programmatically access restaurant menus, categories and menu items.

Base URL: /api/v1

Rate limit: 60 requests / minute per IP.

Locale: Pass ?locale=tr or Accept-Language: tr header to get localised content (default: en).

All endpoints return JSON. Inactive or suspended restaurants return 404.

Authenticating requests

This API is not authenticated.

Menu

Endpoints for fetching public restaurant menus. All endpoints require the restaurant to be active and not suspended.

Full menu

Returns the restaurant info along with all active categories and their items. Response is cached for 5 minutes per locale.

Example request:
curl --request GET \
    --get "http://qrmenu.test/api/v1/menu/demo-restaurant?locale=tr" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json"
const url = new URL(
    "http://qrmenu.test/api/v1/menu/demo-restaurant"
);

const params = {
    "locale": "tr",
};
Object.keys(params)
    .forEach(key => url.searchParams.append(key, params[key]));

const headers = {
    "Content-Type": "application/json",
    "Accept": "application/json",
};


fetch(url, {
    method: "GET",
    headers,
}).then(response => response.json());

    --header "Content-Type: application/json" \
    --header "Accept: application/json"
const url = new URL(
    "http://qrmenu.test/api/v1/menu/demo-restaurant/items"
);

const params = {
    "category": "starters",
    "search": "bruschetta",
